# Юридическая информация и лицензирование

В этом файле подробно описано, как устроено лицензирование в нашем проекте. Если ты добавил в проект свой код, спрайт, музыку или любой другой файл, ты **обязан** указать для него лицензию. Без этого твой вклад не смогут принять.

> **Главное правило:** ты можешь использовать для своего кода или ассета только те лицензии, которые перечислены ниже. Если нужной лицензии нет в списке — свяжись с мейнтейнерами.

## 1. Где лежат тексты лицензий

Все полные тексты лицензий, которые можно использовать, хранятся в папке [`LICENSES/`](LICENSES/). Не нужно копировать текст лицензии в свой файл — достаточно просто сослаться на неё правильным идентификатором.

## 2. Лицензирование кода

### Основная лицензия

**Весь код проекта по умолчанию распространяется под лицензией `AGPL-3.0-or-later` (GNU AGPLv3).** Если для твоего файла подходит эта лицензия, ты можешь ничего не указывать дополнительно — он автоматически будет считаться AGPLv3.

### Если хочешь другую лицензию

Ты можешь указать иную лицензию для своего кода, но **только из тех, что физически лежат в папке `LICENSES/`**.

### Как указать лицензию для файла с кодом

Есть два способа. Выбери **один**, любой удобный.

#### Способ A: REUSE (рекомендуемый)

Просто добавь в самый верх своего файла комментарий со стандартным SPDX-идентификатором. Синтаксис комментария зависит от языка:

```csharp
// SPDX-FileCopyrightText: 2026 Твоё Имя <email>
// SPDX-License-Identifier: AGPL-3.0-or-later
```

```python
# SPDX-FileCopyrightText: 2026 Твоё Имя <email>
# SPDX-License-Identifier: MIT
```

#### Способ B: Файл `.license`

Создай рядом с твоим файлом файл с **таким же именем**, но с расширением `.license`.

Например, если твой код лежит в `MyCoolSystem.cs`, создай файл `MyCoolSystem.cs.license` и напиши внутри:

```
SPDX-FileCopyrightText: 2026 Твоё Имя <email>
SPDX-License-Identifier: MIT
```

### Что писать в SPDX-License-Identifier

Это должен быть **строгий идентификатор** из [официального списка SPDX](https://spdx.org/licenses/). Для лицензий из нашей папки используй:

| Лицензия        | Идентификатор SPDX         |
|-----------------|----------------------------|
| GNU AGPLv3      | `AGPL-3.0-or-later`        |
| MIT             | `MIT`                      |
| Mozilla MPL 2.0 | `MPL-2.0`                  |
| RPL             | `RPL-1.5`                  |
| Starlight       | `MIT-Starlight`            |

## 3. Лицензирование ассетов

**Ассетами** считается графика, звуки, музыка, шрифты — всё, что не является кодом.

### Какие лицензии можно использовать

Только из этого списка. Другие не пройдут валидацию.

| Лицензия                | SPDX-идентификатор    | Что значит кратко                                               |
|-------------------------|-----------------------|-----------------------------------------------------------------|
| Creative Commons BY 3.0 | `CC-BY-3.0`           | Можно всё, но укажи автора                                     |
| Creative Commons BY 4.0 | `CC-BY-4.0`           | То же, но версия 4.0                                           |
| Creative Commons BY-SA 3.0 | `CC-BY-SA-3.0`    | Можно всё, укажи автора, и производные — под той же лицензией  |
| Creative Commons BY-SA 4.0 | `CC-BY-SA-4.0`    | То же, версия 4.0                                              |
| Creative Commons BY-NC 3.0 | `CC-BY-NC-3.0`    | Можно всё некоммерчески, укажи автора                          |
| Creative Commons BY-NC 4.0 | `CC-BY-NC-4.0`    | То же, версия 4.0                                              |
| Creative Commons BY-NC-SA 3.0 | `CC-BY-NC-SA-3.0` | Некоммерчески, укажи автора, производные под той же лицензией  |
| Creative Commons BY-NC-SA 4.0 | `CC-BY-NC-SA-4.0` | То же, версия 4.0                                              |
| Creative Commons BY-ND 3.0 | `CC-BY-ND-3.0`    | Можно всё, но без изменений, укажи автора                      |
| Creative Commons BY-ND 4.0 | `CC-BY-ND-4.0`    | То же, версия 4.0                                              |
| Creative Commons BY-NC-ND 4.0 | `CC-BY-NC-ND-4.0` | Некоммерчески, без изменений, укажи автора                     |
| Creative Commons Zero 1.0 | `CC0-1.0`         | Общественное достояние. Можно всё, не указывая никого.         |
| MIT                     | `MIT`                 | Почти всё можно, укажи автора и лицензию                       |
| Custom                  | `Custom`              | Особая лицензия (см. ниже)                                     |

Если у тебя особая лицензия, которой нет в SPDX (например, автор разрешил использовать только в этом проекте), ставь `Custom`. Тогда в поле `copyright` (см. ниже про RGA) **обязательно** напиши, что это за лицензия и дай на неё ссылку.

## 4. Как указать лицензию для ассетов: RGA (Robust Generic Attribution)

Для **всех** ассетов мы используем формат **RGA**. Это просто YAML-файл с именем `attributions.yml`, который лежит в той же папке, что и твои файлы.

### Куда класть файл

Файл должен называться `attributions.yml` и лежать **в той же папке**, что и сами ассеты. Он действует на все файлы в этой папке, **но не на подпапки**. Для подпапок создаются свои `attributions.yml`.

### Структура записи

Каждая запись (entry) в YAML описывает один или несколько файлов. Поля записи:

| Поле         | Обязательно | Значение                                                                                                                                                                                                                                                                                                            |
|--------------|:-----------:|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `files`      | Да          | Массив имён файлов (с расширениями), к которым относится эта запись. Можно использовать `*` как wildcard. Например, `"*.png"` — все PNG в папке, `"goblin_idle_*.ogg"` — все OGG, начинающиеся с `goblin_idle_`.                                                                                                   |
| `license`    | Да          | SPDX-идентификатор из списка выше. Например, `"CC-BY-SA-4.0"`. Если лицензия особая, пиши `"Custom"`.                                                                                                                                                                                                                |
| `copyright`  | Да          | Информация об авторских правах: кто создал, что изменено. Если лицензия `Custom`, здесь **должна быть** информация о лицензии и ссылка на неё.                                                                                                                                                                       |
| `source`     | Да          | URL, откуда скачан оригинал. Если файл сделан тобой с нуля, можно указать ссылку на твой Pull Request, которым ты его добавляешь. Если это чужая работа, укажи оригинальный источник. Если файл сильно изменён — дай ссылку на изменённую версию, а в `copyright` укажи ссылки на оригиналы.                       |

### Пример готового `attributions.yml`

```yaml
# attributions.yml — пример
# Этот файл лежит, например, в Resources/Audio/Music/

- files: ["thunderdome.ogg"]
  license: "CC-BY-NC-SA-3.0"
  copyright: "-Sector11 by MashedByMachines. Converted from MP3 to OGG."
  source: "https://www.newgrounds.com/audio/listen/312622"

- files: ["endless_space.ogg"]
  license: "CC-BY-3.0"
  copyright: "Endless Space by SolusLunes. Converted from MP3 to OGG."
  source: "https://www.newgrounds.com/audio/listen/67583"

- files: ["my_own_track.ogg"]
  license: "CC-BY-SA-4.0"
  copyright: "Created by YourName for Space Station 14."
  source: "https://github.com/space-wizards/space-station-14/pull/12345"

- files: ["*.wav"]
  license: "CC0-1.0"
  copyright: "Various authors. All sounds from freesound.org, released under CC0."
  source: "https://freesound.org/"
```

### Важные правила RGA

1. `attributions.yml` **обязательно** должен описывать **все** файлы в папке. Нельзя оставить какой-то файл без записи.
2. Один файл не может быть описан в двух записях одновременно.
3. Если у файла особая лицензия (`Custom`), поле `copyright` **должно** содержать условия и ссылку.
4. Всегда проверяй, что `SPDX-идентификатор` написан без ошибок. Идентификаторы регистрозависимы: `CC-BY-4.0` — правильно, `cc-by-4.0` — неправильно.
5. Не используй лицензии, не перечисленные в разделе 3, если это не согласовано отдельно.

## 5. Краткий алгоритм: что делать, когда добавляешь файл

### Я добавляю код
1. Открой папку `LICENSES/` и посмотри, есть ли там нужная лицензия (по умолчанию все используют `AGPL-3.0-or-later`).
2. Добавь в начало файла комментарий:
   ```csharp
   // SPDX-License-Identifier: AGPL-3.0-or-later
   ```
3. Или создай рядом файл `.license` с той же строкой.
4. Готово.

### Я добавляю ассет(ы)
1. Положи файлы в нужную папку.
2. Если в папке уже есть `attributions.yml`, добавь в него новую запись.
3. Если файла `attributions.yml` ещё нет, создай его.
4. Заполни поля `files`, `license`, `copyright`, `source` по правилам выше.
5. Убедись, что лицензия есть в списке разрешённых (раздел 3). Если нет — спроси мейнтейнеров.

## 6. Часто задаваемые вопросы

**В: Что писать в `copyright`, если я автор?**
О: Просто укажи своё имя/ник и год. Например: `Copyright (c) 2026 YourName`.

**В: Я взял чужой код под MIT, что делать?**
О: В папке `LICENSES/` есть `MIT.txt`. Просто добавь SPDX-идентификатор `MIT` в свой файл. Убедись, что ты соблюдаешь условия MIT (обычно это требует сохранить копирайт оригинального автора).

**В: SPDX-идентификатор и то, что в названии файла в LICENSES — это одно и то же?**
О: Почти. SPDX-идентификатор используется для машинной обработки. Название файла в `LICENSES/` должно совпадать с SPDX-идентификатором. Например, лицензия `AGPL-3.0-or-later` лежит в файле `AGPL-3.0-or-later.txt`.

**В: Я хочу добавить шрифт с лицензией SIL OFL. Можно?**
О: Нет, этой лицензии пока нет в списке разрешённых. Нужно обсудить с мейнтейнерами и сначала добавить её в список разрешённых.

**В: Что если я не укажу лицензию?**
О: Твой Pull Request не примут до тех пор, пока ты не добавишь корректную информацию о лицензировании.
