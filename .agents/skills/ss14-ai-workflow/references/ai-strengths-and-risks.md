# AI Strengths And Risks

## AI Is Useful For

- explaining unfamiliar systems;
- drafting repetitive component/system/prototype scaffolding;
- writing focused tests from an existing API;
- summarizing diffs and review risks;
- turning docs into checklists.

## AI Is Risky For

- inventing architecture without repo context;
- handling prediction edge cases from memory;
- guessing current .NET or Robust APIs;
- adding hardcoded one-off branches;
- silently copying stale upstream examples.

## SS14-ART-CORE Rule

AI output is a draft until verified against local code, `.agents/rules`, and focused validation.
